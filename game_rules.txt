Общая концепция портала
●
●
●
Цель: Создать атмосферный, простой и устойчивый веб-портал, который служит
«базой данных» для настольной игры.
Стиль: Нуар, тёмная тема, минимализм. В качестве акцентного цвета используется
красный для важных кнопок и обозначения ошибок.
Адаптивность: Портал должен корректно отображаться как на десктопных
компьютерах, так и на мобильных устройствах.
Пользовательский путь и описание экранов
Процесс взаимодействия с порталом разделён на несколько ключевых экранов и этапов.
Шаг 0: Физический объект (Начало игры)
Взаимодействие начинается не с портала, а с физической визитной карточки, которая
находится в игровой коробке.
●
Лицевая сторона: Логотип, надписи "Lux in Tenebris" и "ИГРА НАЧИНАЕТСЯ
ЗДЕСЬ".
●
Оборотная сторона: QR-код с подписью "Отсканируйте, чтобы принять вызов".
Экран 1: Лендинг (после сканирования QR-кода)
Это первый экран, который видит пользователь. Его задача — погрузить в атмосферу.
●
Основной элемент: Полноэкранное видео "входящего вызова" от персонажа по
имени Веспер Уэйнрайт.
○
Видео запускается автоматически, но без звука.
○
Звук включается после первого клика пользователя в любом месте экрана
(это стандартное ограничение для мобильных браузеров).
●
●
Элементы поверх видео (Оверлей): Две кнопки:
○
[ДА, принимаю дело]
○
[НЕТ]
●
●
Логика кнопок:
○
При нажатии [НЕТ] → появляется текст "Ривертон останется без ответа" и
кнопка "Вернуться". Это "шуточная" концовка, которая не позволяет
продолжить игру.
○
При нажатии [ДА] → появляется кнопка [Войти], которая ведёт на экран
входа.
●
●
Резервный вариант (Фолбэк): Если по какой-то причине видео не загружается,
вместо него отображается статичное изображение (постер) и кнопка «Начать
звонок», которая имитирует начало видео.
Экран 2: Логин
Стандартный экран для ввода учётных данных.
●
Элементы:
○
○
○
Поле для ввода логина.
Поле для ввода пароля.
Кнопка "Войти" (или аналогичная).
●
●
●
Данные: Используется одна пара логина и пароля, которая предоставляется в
игровой коробке.
Логика:
○
○
При успешном вводе данных пользователь переходит в "Личный кабинет".
При ошибке — появляется короткое сообщение об ошибке, без уточнения,
что именно было введено неверно (логин или пароль).
●
Экран 3: Личный кабинет (Главная панель)
Это основной интерфейс, где игрок проводит большую часть времени.
●
Навигация: Панель навигации, расположенная сверху или слева. Она содержит 4
основных раздела:
1. МАТЕРИАЛЫ ДЕЛА
2. БАЗА ЗНАНИЙ
3. АУДИО-АРХИВ
4. АТТЕСТАЦИЯ
●
Ниже приведено детальное описание каждого раздела.
Описание разделов Личного кабинета
1. Раздел «МАТЕРИАЛЫ ДЕЛА»
●
Что находится: Здесь хранятся все улики по делу: тексты, отчёты, сканы
документов, фотографии.
●
●
●
Стартовое состояние: Раздел пуст или содержит только один документ:
"Входящие: Дело № DT-2025-08-30-SB".
Логика: Доступ к материалам открывается поэтапно. Новые улики появляются
после успешного прохождения Тестов №1, №2 и №3.
UX-элементы:
○
○
○
У документов есть пометки: «новое» и «просмотрено».
Есть фильтр для сортировки материалов по этапам их получения
(Старт/К1/К2/КЗ).
При добавлении новых материалов появляется всплывающее уведомление,
а сам раздел в меню навигации подсвечивается.
●
2. Раздел «БАЗА ЗНАНИЙ»
●
●
Что находится: Справочная информация, которая доступна с самого начала и не
меняется.
Содержимое:
○
Протокол DT-7: Полевая дактилоскопия (версия с "ловушкой").
○
Справка: препарат «Сертофин».
○
Досье: город Ривертон.
○
Справка: галерея «Хранилище».
●
3. Раздел «АУДИО-АРХИВ»
●
●
●
●
Структура: Раздел состоит из двух вкладок (табов).
Вкладка 1: «Видеодопросы»
○
Содержимое: 6 видеозаписей допросов.
○
Доступ: Все видео доступны сразу после входа в личный кабинет.
Вкладка 2: «Голосовые сообщения»
○
Элементы: Поле для ввода кода и кнопка «Прослушать».
○
Логика: Пользователь вводит цифровые коды, найденные в игре. Если код
верный, открывается соответствующее аудиосообщение. Прослушанные
сообщения остаются в списке для повторного доступа.
●
4. Раздел «АТТЕСТАЦИЯ»
●
сюжету.
●
Логика:
Что находится: Система тестов для проверки знаний игрока и продвижения по
1. Тесты (№1, №2, №3): Три промежуточных теста и один финальный отчёт.
■ Условия: Для успешного прохождения нужно ответить на 100%
вопросов правильно.
■ В случае неудачи: Кнопка «Попробовать снова» становится
неактивной на 30 секунд для предотвращения подбора ответов
(анти-брутфорс).
■ В случае успеха:
■ После Теста №1 → открываются материалы Конверта №1.
■ После Теста №2 → открываются материалы Конверта №2 и
появляется блок «Санкция на осмотр локаций».
■ После Теста №3 → открываются материалы Конверта №3 и
появляется кнопка «Подать финальный отчёт».
■
2.
3. Блок «Санкция на осмотр локаций» (после Теста №2):
■ Элементы: Экран с 6 карточками локаций (Розалия, Эван, Маркус и
др.).
■ Логика: Игрок должен выбрать ровно две локации для осмотра.
После подтверждения выбора две соответствующие фото-страницы
становятся доступны, а остальные четыре блокируются навсегда для
текущей сессии. Выбор сохраняется на сервере.
4.
5. Форма «Финальный отчёт» (после Теста №3):
■ Элементы: Экран в виде официального бланка с вопросами.
■ Динамика: Форма интерактивна — ответ на вопрос №2 меняет
варианты ответов для вопросов №3 и №4 без перезагрузки
страницы.
■ Отправка: После заполнения игрок нажимает «Отправить отчёт в
архив». Появляется модальное окно с предупреждением «Действие
необратимо». После подтверждения данные отправляются на сервер
для проверки.

Экран 4: Эпилог (финальный экран)
Этот экран пользователь видит после отправки финального отчёта. Содержимое зависит
от правильности его выводов.
●
Если комбинация ответов верная (Победа):
○
Что находится: Показывается страница «Эпилог Победы». На ней — видео
«Исповедь Розалии» и финальный текст.
●
●
Если комбинация ответов неверная (Неудача):
○
Что находится: Показывается страница «Эпилог Неудачи» с текстовым
описанием последствий.
